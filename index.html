以太网的帧格式

发表于 2019-08-20 | 分类于 网络

参考：《图解TCP/IP》人民邮电出版社 p92~95

​ 以太网帧前端有一个叫做前导码的部分，表示一个以太网帧的开始，也是对端网卡能够确保与其同步的标志。前导码由7个字节组成，后面跟着一个字节的帧开始符。传输一个字节时由最低位开始传输(LSB)。

以太网帧本体的前端是以太网的部首，总共14个字节，分别为6字节的目标MAC地址、6字节的源MAC地址以及2字节的上层协议类型。

​ 帧头后是数据，数据帧最大能容纳的数据范围是46~1500个字节。帧尾接FCS(Frame Check Sequence,帧检查序列)的四字节。FCS保存着整个帧除以生成多项式的余数。

​ 在IEEE802.3的帧格式中附加的LLC与SNAP就是由逻辑链路控制的部首信息。

电子邮件的相关协议与数据格式

发表于 2019-08-19 | 分类于 网络

参考：《图解TCP/IP》人民邮电出版社 p260~267

电子邮件的数据格式 MIME(Multipurpose Internet Mail Extensions)

​ 早期的电子邮件只能处理文本格式的邮件，后来扩展到MIME后可以发送静态图像、动画、音频、程序等各种形式的数据。鉴于MIME规定了应用消息的格式，因此在OSI参考模型中其相当于第6层的表示层。

电子邮件的通讯协议

1、SMTP(Simple Mail Transfer Protocol)

​ SMTP是发送电子邮件的协议。使用TCP的25号端口。发送端将邮件通过该协议发送到邮件服务器。

​ 早期电子邮件是发送端主机和接收端主机通过SMTP直接传输，这就需要两端的主机同时处于开机状态才能正常收发邮件。

2、POP(Post Office Protocol)

​ POP协议是接收端从邮件服务器获取邮件的协议。发送端邮件通过SMTP协议发送到一直处于插电状态的POP服务器，客户端再根据POP协议从POP服务器接收邮件。这个过程中需要进行用户验证以防止他人盗窃邮件。

​ POP中邮件由客户端进行管理。在POP中无法下载某个特定的附件。如要确认某个附件就必须将所有附件下载下来。其未读信息与邮件分类也都是在客户端中管理。POP协议在邮件由客户端接收完成后服务器端会删除邮件，在该协议最新版本POP3中可以做到只下载邮件，服务器端不删除。

3、IMAP(Internet Message Access Protocol)

​ IMAP与POP类似，也为接收电子邮件的协议。与POP不同的是IMAP中的邮件由服务器进行管理。可以不必从服务器上下载整个邮件也可以阅读，对于“已读/未读”及分类管理也都是在服务器端完成。

快乐辉光管

发表于 2019-08-17 | 分类于 瞎玩

​ 很早以前看了命运石之门，对里面的世界线变动率探测仪真的是一见钟情。辉光管那种独特的橙色光，还有那字体真的太美了。趁着大四闲的时候想自己搞一发。

​ 查了下淘宝，立式的管子真的是贵的令人发指，100多一根，打扰了。退而求其次，用15一个的QS27来做了。买了6个管子，加底座。准备做个钟。

​ 其实原理十分简单。管子总共11个脚，其中一个是公共地，其他十个脚对应数字0~9，给个170V电压和限流电阻对应的数字就亮了。

​ 整个系统准备用12V的电源供电，使用DCDC升压得到170V。开始使用34063来做升压，发现真的不好用，因为这个芯片的占空比是固定的，通过控制脉冲的数量来控制输出电压。对于12V~170V这种大压差的升压根本没法用。最后改用UC3845，典型的BOOST升压电路，很容易就升上去了(当时直接用洞洞板做的，忘留图了)接上辉光管后会有一定的压降，最后稳定在160V左右。

​ 逻辑部分使用stm32f103c8t6作为主控。由于不想用动态显示的方案，所以使用38译码器加上锁存器的方案。由于有10个数字，所以需要两片38译码器组合使用，锁存器也得用两片来组合使用。锁存器输出接上高压三极管作为开关。这种方案最大的问题就是线太多了。如果直接用移位寄存器线会少很多。由于想体现那种复古的感觉，所以器件能用直插的就用直插的。

一、通过函数指针创建线程

​ 定义函数counter()，传入两个参数：第一个表示线程的ID，第二个表示这个函数要循环的次数。

通过std:thread 启动多个线程。例如创建线程1，以参数1和6作为creater的参数:

thread类的构造函数是个可变参数模板，可以接受任意数目的参数。

​ 其中，join函数的用处是阻塞主调线程(即main函数的主线程)，等待被调线程(t1与t2)终止，再由主调线程回收被调线程的资源，继续运行。在本例中的用途就是防止被调线程未执行完，而main线程先结束，导致程序停止执行。

二、通过函数对象创建线程

​ 使用函数对象，可向函数对象类添加成员变量，并可以采用任何方式初始化和使用这些变量。定义一个Counter类，其中有两个成员变量：一个表示ID，一个表示循环次数。成员变量通过类构造函数初始化。

int main()
{
    //Using uniform initialization syntax C++11后推出的新特性 统一初始化语法 初始化用{ }就完事了
    thread t1{ Counter{1,6} };
    //Using named varibale 实例化类
    Counter c(2, 12);
    thread t2(c);
    //Using temporary 
    thread t3(Counter(3, 10));

    //等待线程结束
    t1.join();
    t2.join();
    t3.join();
}

​ 使用t3那种初始化方法的时候需要注意，如果函数对象构造函数不含任何参数，则上面写的方法不能正常工作。例如下面这种写法就会报错。编译器会将thread t3(Counter());视作t1函数的声明，这个函数返回thread对象。

​ 尽量使用统一初始化语法，即第一种方法。如果编译器不支持，则必须添加额外的圆括号来防止编译器将这条语句解释为函数声明。
